从零开始搭建 nas

G·D·D2024/10/07nas

winnas

虽然 Linux 在服务器领域占据主导地位，但相比于 Linux ，win 更符合大多数人的使用习惯。

优势

  • 易用性:
    • 符合用户习惯: 对于熟悉 Windows 的用户来说，上手和管理 Windows 服务器更加容易，无需学习新的命令行操作。
    • 图形化界面: 图形化界面 (GUI) 使得服务器管理更加直观，出现问题可以快速定位和解决，无需依赖命令行。
    • 服务管理: 安装服务端软件后，可以快速启动服务并以后台形式运行，方便管理。
  • 文件系统兼容性:
    • 直接读取硬盘: Windows 可以直接读取 NTFS 格式的硬盘，方便数据迁移和共享。
  • 软件生态:
    • 丰富的商业软件: Windows 拥有丰富的商业软件生态，尤其是一些行业特定的软件，可能只提供 Windows 版本。

缺陷

  • 资源占用高:
    • 内存占用: Windows 系统本身以及图形化界面会占用较多内存资源，对于资源有限的服务器来说，这可能是个问题。
    • 功耗： Windows 系统的平均待机功耗高于 linux 系统，对于7x24小时开机的设备来说，电费不能忽视。
    • Docker 性能: 在 Windows 上运行 Docker 容器性能不如 Linux，因为 Windows 不是原生支持 Docker 的 Linux 内核。
  • 应用选择受限:
    • 开源软件支持: 许多优秀的开源服务器软件优先支持 Linux，Windows 下的选择相对较少。
    • 更新和维护: 一些开源软件在 Windows 下的更新和维护可能不如 Linux 及时。
  • 安全性:
    • 攻击面更大: Windows 系统本身以及其上运行的软件可能存在更多安全漏洞，需要更加注意安全防护。

    • 远程桌面(RDP): 其他服务坏了可以用 RDP 修。
    • 影音库: win 一定是最早支持最新的显卡驱动的，对转码有需求的

硬件选择思路

以下只考虑用消费级硬件 DIY

  • 功耗

    • CPU： 选用高能效的cpu，至少为4代酷睿之后，4代是睿频技术成熟的一代。3代酷睿之前的尽量不要选用！ C状态(C-states)
    • 内存： 单根比一根省电，双通道的提升在服务器用途中不明显。选单跟大容量，后期升级也方便。
    • 硬盘： HDD比SSD更耗电。另外 HDD 除了便宜没有优势。
    • 电源： 80plus认证，建议铜牌以上。

  • 噪音

    • 风扇： 富哥上猫扇，平民上利民。一般情况转速越低越安静，所以14cm比12cm更合适在保持风量一样的情况转速更低。老主板(三代酷睿之前)请检查风扇插针是4pin还是3pin，3pin请酌情选用。另外，在bios里调转速曲线
    • HDD： 加 SSD 做缓存 可以缓解炒豆子的声音。因为顺序读写的噪音比多次读写的噪音要小。
    • CPU散热器： 低功耗U 一般下压都没问题，双塔可以只装中间一个风扇。另外，在bios里调转速曲线

  • 拓展性

    • pcie： z板带有pcie拆分，itx拓展性低且不需要拆分不用选z板，matx和atx建议z板。
